E-Shop POC Application

Welcome to the technical documentation of the E-Shop Proof of Concept application. Below you will find links to the OpenAPI specifications for each microservice in the application, along with a brief description of each service and further architectural documentation.

High Level Architecture

API Definitions

  • AuthService

    Handles authentication and authorization, including login, logout, and token management.

    • API Documentation
    • OpenAPI YAML Specification

  • UserService

    Manages user profiles, registration, and account updates.

    • API Documentation
    • OpenAPI YAML Specification

  • ProductService

    Manages the catalog of digital products, including creation, updating, and deletion of products.

    • API Documentation
    • OpenAPI YAML Specification

  • CartService

    Allows users to add items to their shopping cart, view items, modify quantities, delete items, empty the cart, and proceed to checkout.

    • API Documentation
    • OpenAPI YAML Specification

  • OrderService

    Processes orders and maintains order status. Provides endpoints for viewing orders and order details.

    • API Documentation
    • OpenAPI YAML Specification

  • PaymentService

    Manages payment methods and processes payments. Allows users to retrieve available payment methods and admins to manage them.

    • API Documentation
    • OpenAPI YAML Specification

  • LicenseService

    Generates and manages digital licenses for purchased products. Admins can manage license templates and view generated licenses.

    • API Documentation
    • OpenAPI YAML Specification

  • NotificationService

    Sends notifications (e.g., emails) to users. Admins can list and filter sent notifications.

    • API Documentation
    • OpenAPI YAML Specification

  • TrackingService

    Collects and records events for analytics and monitoring. Admins can list and filter recorded events.

    • API Documentation
    • OpenAPI YAML Specification
