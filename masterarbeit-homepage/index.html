High Level Architecture

Below is a high-level overview of the system architecture, illustrating how the microservices interact with each other and external components.

AuthService

Handles authentication and authorization: login, logout, and token management.

  • API Documentation
  • OpenAPI YAML

UserService

Manages user profiles, registration, and account updates.

  • API Documentation
  • OpenAPI YAML

ProductService

Manages digital product catalog: creation, updating, and deletion of products.

  • API Documentation
  • OpenAPI YAML

CartService

Allows users to add items to cart, view, modify, delete items, and proceed to checkout.

  • API Documentation
  • OpenAPI YAML

OrderService

Processes orders, maintains order status, and provides order details.

  • API Documentation
  • OpenAPI YAML

PaymentService

Manages payment methods and processes payments. Offers endpoints for available payment methods and admin management.

  • API Documentation
  • OpenAPI YAML

LicenseService

Generates and manages digital licenses for purchased products. Admins can manage license templates and view generated licenses.

  • API Documentation
  • OpenAPI YAML

NotificationService

Sends notifications (e.g., emails) to users. Admins can list and filter sent notifications.

  • API Documentation
  • OpenAPI YAML

TrackingService

Collects and records events for analytics and monitoring. Admins can list and filter recorded events.

  • API Documentation
  • OpenAPI YAML
